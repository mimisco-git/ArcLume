ArcLume Connected Mode

Prepared for future authenticated Arc House connection without breaking the static site.

What this page is for

ArcLume V8 includes an optional connected mode. It is disabled by default and the main site remains fully functional without it.

Auth-ready

Static mode today

The dashboard can already show live Arc wallet data and the public Arc House feed without any private integration.

Connected mode later

When a supported Arc House authentication or profile flow is available, you can point ArcLume at your connector through runtime-config.js.

Files included

runtime-config.js, runtime-config.example.js, auth/callback.html, and api/cloudflare-worker.example.js.

Important boundary

Do not place secrets in the static site. Keep any token exchange or profile retrieval on your worker or backend.

Suggested setup

A minimal safe path for future connected mode.

  1. Deploy the static ArcLume site to GitHub Pages as usual.
  2. Deploy your worker or serverless adapter separately.
  3. Update runtime-config.js with your public auth and profile endpoints.
  4. Keep all credentials server-side only.
  5. Use auth/callback.html for future redirect handling if your supported auth flow needs it.
